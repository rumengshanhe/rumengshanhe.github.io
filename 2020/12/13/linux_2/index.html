linux介绍

定义

Linux–是一种类unix操作系统，确切的说是指操作系统的内核(kernel)。Linux是自由软件和开放源代码软件发展中最著名的例子。

Linux 发行版本

  • redhat : 公司运营，提供完整的解决方案，更专业
  • fedora : 和Ubuntu的包管理不一样，Fedora对新软件及新技术吸纳比较快
  • debian : 最遵循GNU规范操作系统，Ubuntu就是基于Debian的
  • Ubuntu : 国内乃至全球热门的Linux发行版，安装简单，社区活跃
  • Centos : 特点是相当稳定，版本更新紧跟RedHat，非常适合作为服务器操作系统使用
  • Suse linux : 德国著名linux，被称为“最华丽的Linux发行版”
  • 红旗Linux : 由中国中科红旗软件技术有限公司开发,方便国内的用户使用

linux基本命令

分类

  • 内部命令：就是shell自带的一些命令，而这些命令是不可以卸载。
  • 外部命令：就是用户自己安装的一些命令。而这些命令是有路径。

区分内部命令和外部命令

type + cmd

[root@localhost /]# type fdisk

fdisk is hashed (/sbin/fdisk) //表示外部命令

[root@localhost /]# type cd

cd is a shell builtin //表示内部命令

查看命令别名

[root@localhost /]# type ls

ls is aliased to ‘ls –color=tty’

[root@localhost /]# which ls

alias ls=’ls –color=tty’

/bin/ls

输出命令

输出命令既是内部命令也是外部命令

[root@localhost /]# echo hello

hello

[root@localhost /]# type echo

echo is a shell builtin

[root@localhost /]# which echo

/bin/echo

tab 键

作用是自动补齐

  • 文件名自动补齐
  • 命令自动补齐

如果按一次tab键没有反应，有下列情况：

1、 没有该文件或者命令

2、 可能匹配多个文件或者命令 则再按一次。

man—帮助命令

man命令是Linux下的帮助指令，通过man指令可以查看Linux中的指令帮助、配置文件帮助和编程帮助等信息。

  • Format   :   man + cmd

    案例 : man 7 man

  • Option:
  • a：在所有的man帮助手册中搜索；
  • f：等价于whatis指令，显示给定关键字的简短描述信息；
  • P：指定内容时使用分页程序；
  • M：指定man手册搜索的路径。
参数（代号 含义 代表内容
1 Commands 表示基本命令
2 System calls 表示kernel说明信息
3 Library calls 表示C库或者第三方图形库
4 Special files 表示设备类文件说明（字符设备、块设备）
5 File formats and conventions 表示文件格式说明文档
6 Games 表示游戏开发相关说明文档
7 Conventions and miscellaneous 表示网络相关说明文档
8 System management commands 表示管理员命令
9 Kernel routines 内核相关说明文档

help

help 是非常简单的命令，而且不经常使用。对于外部命令，可以使用cmd --help来获取帮助。

案例：ls --help

对于内部命令，可以直接使用help cmd格式还获取命令帮助。

案例： help cd

info

info命令是Linux下info格式的帮助指令。

案例：info ls

  • Option：
  • d：添加包含info格式帮助文档的目录；
  • f：指定要读取的info格式的帮助文档；
  • n：指定首先访问的info帮助文件的节点；
  • o：输出被选择的节点内容到指定文件。

ls

显示当前工作目录的所有信息

  • Format:

    ls [OPTION] [FILE]

  • Option :

    -a : 表示显示所有信息(包括 . .. 和隐藏文件或者目录)

    .  :  表示当前工作目录

    ..  :  表示上级目录

    .filename  :  表示隐藏文件

    -i  :  显示文件索引号
    inode节点号（文件索引号）是计算机识别文件的唯一标识

    -l  :  表示长格式显示文件信息

案例：[root@localhost test]# ls -l

总计 16

-rwxr-xr-x 1 root root 4729 11-29 20:18 a.out

-rw-r–r– 1 root root 6 11-29 20:13 a.txt

-rw-r–r– 1 root root 78 11-29 20:18 hello.c

  • 第一列：表示文件类型和权限

第一位表示文件类型

- : 表示普通文件

d : 表示目录

c : 表示字符设备类文家

b : 表示块设备文件

l : 表示链接文件

p : 表示管道文件

s : 表示套接子文件

第二到第十位表示权限

r : 表示只读

w : 表示只写

x : 表示可执行

- : 表示无权限

三位为一组，共分为三组

第一组：表示属主权限

第二组：表示属组权限

第三组：表示其他用户权限

  • 第二列：表示文件硬连接数
  • 第三列：表示文件属主 – 文件拥有者
  • 第四列：表示文件属组 – 文件拥有组
  • 第五列：表示文件大小（字节）
  • 第六列：表示时间
  • 第七列：表示文件名

文件系统

在linux下，所有的设备皆文件，所有的文件都是从根开始。

文件目录详解

  • / : 表示根
    etc ： 表示配置文件目录
    sbin ： 表示存放管理员命令可执行文件目录
    bin ： 表示基本命令目录
    home ： 表示普通用户家目录
    media ： 表示挂在U盘、光盘目录
    opt : 表示扩展包存放路径
    proc ：通过proc可以得到系统的信息，并可以改变内核的某些参数

    cat /proc/cpuinfo ： 查看CPU信息

    cat /proc/meminfo ： 查看内存大小
    sys ： 表示虚拟目录
    dev ： 表示设备类目录
    lib ： 表示库目录
    动态库： .so
    静态库： .a
    mnt : 表示挂载目录
    root ：表示管理员家目录

    /root/Desktop/ ： 表示桌面

查看系统

  • [root@localhost /]# lsb_release -a

    LSB Version:</br>

    :core-3.1-ia32:core-3.1-noarch:graphics-3.1-ia32:graphics-3.1-noarch

    Distributor ID: RedHatEnterpriseServer //系统名

    Description: Red Hat Enterprise Linux Server release 5.5 (Tikanga)

    Codename: Tikanga

  • [root@localhost /]# uname -a

    Linux localhost.localdomain 2.6.18-194.el5 #1 SMP Tue Mar 16 21:52:43 EDT

    2010 i686 i686 i386 GNU/Linux

    Linux : 表示内核

    localhost.localdomain ：表示主机名

    2.6.18-194.el5 ：表示内核版本

    #1 SMP Tue Mar 16 21:52:43 EDT 2010 : 表示发布时间

    i686 i686 i386 ：表示系统位数

    GNU/Linux ：表示操作系统名

  • [root@localhost /]# uname -r : 查看内核版本

    2.6.18-194.el5

    表示 : x.y.z

    x : 表示主版本号，除非系统架构发生改变

    y : 表示次版本号

    奇数：测试版本

    偶数：稳定版本

    z : 表示释放版本

  • [root@localhost /]# uname -m : 系统位数

    i686    ：表示32位系统</br>

    x86_64 : 表示64位系统
